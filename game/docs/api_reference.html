File: game/main.py

Purpose

Provides the local client entry point. It applies Panda3D settings, creates GameManager, and starts the engine loop.

Responsibilities

  • Define immutable startup settings with AppConfig.
  • Apply Panda3D PRC configuration before engine construction.
  • Instantiate the generated GameManager.
  • Return a process exit code after shutdown.

Dependencies

argparse, dataclasses, pathlib, sys, panda3d.core.loadPrcFileData, and game_manager.GameManager.

Dependents

Command-line execution and future launch scripts depend on this entry point.

Future Multiplayer Impact

This file should remain client-only. A future dedicated server can use a separate entry point that constructs shared world systems without Panda3D window configuration.

File: game/game_manager.py

Purpose

Owns the Panda3D local client shell and provides the root lifecycle hooks that future systems will attach to.

Responsibilities

  • Create and own Panda3D ShowBase.
  • Register the central per-frame update task.
  • Store startup configuration and project paths.
  • Keep future systems coordinated without becoming world authority.

Dependencies

pathlib.Path, typing.Any, direct.showbase.ShowBase.ShowBase, and panda3d.core.ClockObject.

Dependents

game/main.py constructs this class during local client startup.

Future Multiplayer Impact

Remains client-only. It will eventually send player action requests and consume world events, while authoritative world mutation belongs to WorldManager and later a dedicated server.

Package Boundary Files

game/player/__init__.py

Marks the player package. Future code owns first-person movement, mouse-look camera control, and collision queries. Player code must request world changes through WorldManager.

game/world/__init__.py

Marks the world package. Future code owns central world authority, world events, block access rules, placement validation, and destruction validation.

game/chunks/__init__.py

Marks the chunks package. Future code owns serializable chunk data, loading, unloading, and network-ready chunk payload shape.

game/terrain/__init__.py

Marks the terrain package. Future code owns deterministic seed-based terrain and biome generation.

game/inventory/__init__.py

Marks the inventory package. Future code owns inventory slots, hotbar selection, and item validation separately from world mutation.

game/rendering/__init__.py

Marks the rendering package. Future code owns chunk meshes, scene nodes, lighting, and render-only consumption of world events.

game/saving/__init__.py

Marks the saving package. Future code owns persistence of world metadata, dirty chunks, player state, inventory state, and settings.

AppConfig

Filegame/main.py
PurposeStores startup settings used before Panda3D starts.
LifecycleCreated once in main() and passed into create_game().
Dependenciesdataclasses.dataclass.
Propertieswindow_title, window_width, window_height, target_fps, show_frame_rate_meter, sync_video, cursor_hidden.
PerformanceImmutable and tiny; no meaningful runtime cost.
SerializationNot currently serialized. May later be mirrored by user settings.
MultiplayerClient-specific. Server config should live in a separate type.

GameManager

Filegame/game_manager.py
PurposeTop-level lifecycle manager for the local Panda3D client.
LifecycleConstructed once by main.create_game(); lives until the app exits or close() is called.
DependenciesShowBase, ClockObject, project root, and AppConfig-compatible settings.
Propertiesproject_root, app_config, elapsed_time, is_shutting_down.
PerformanceCurrent update cost is O(1). Future systems must keep per-frame work budgeted.
SerializationDoes not serialize game state. Future SaveManager owns persistence.
MultiplayerClient coordinator only; not canonical world authority.

create_game(config)

PurposeApplies config, imports GameManager, and constructs the game.
Parametersconfig: AppConfig.
ReturnsA GameManager instance.

main()

GameManager.__init__(project_root, app_config)

PurposeInitializes the Panda3D client shell.
Parametersproject_root: Path, app_config: Any.
ReturnsNone.
Side EffectsCreates Panda3D app state, sets background color, disables default mouse control, and registers the update task.

GameManager.update(task)

PurposeCentral per-frame hook for the client shell.
Parameterstask, supplied by Panda3D's task manager.
Returnstask.cont while active or task.done during shutdown.
Side EffectsAccumulates elapsed time.
ExceptionsNo expected exceptions.

GameManager.close()
