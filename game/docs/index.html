Minecraft-Inspired Voxel Sandbox

Developer documentation for a playable Panda3D voxel sandbox with chunked world streaming, deterministic terrain, block editing, inventory, saving, day/night lighting, and a client-first architecture prepared for later dedicated servers.

Project Overview

The game is organized around explicit managers. Player input and rendering run on the client, while world mutation is centralized in WorldManager so multiplayer authority can be introduced later without rewriting gameplay systems.

Initialization Flow

python game/main.py
  |
  |-- build AppConfig
  |-- apply Panda3D PRC settings
  |-- construct GameManager
  |-- GameManager creates managers
  |-- Panda3D task loop starts

Engine Architecture

GameManager
|-- InputManager
|-- WorldManager
|   |-- ChunkManager
|   |-- TerrainGenerator
|   `-- SaveManager
|-- RenderManager
|-- InventoryManager
`-- PlayerController
